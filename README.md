# Сорванцы — Max Bot

[Бот](https://max.ru/id590849202576_bot) лояльности для мессенджера [Max](https://max.ru) — магазин детской одежды **Сорванцы**.

Покупатель сканирует QR-код на кассе, регистрируется в боте и получает персональную скидку. Продавец сканирует QR-код покупателя, видит профиль и активные купоны. Владелец управляет персоналом и рассылками прямо из мессенджера.

---

## Что умеет бот

### Для покупателей

- **Регистрация** — сканирование QR-кода на кассе или переход по ссылке запускает регистрацию. Покупатель получает скидку сразу после первого нажатия «Старт».
- **Анкета** — опционально: имя, дата рождения, данные о детях (имя, пол, дата рождения). Несколько детей — поддерживается.
- **Карточка скидки** — QR-код с процентом скидки, показывается кассиру с экрана телефона.
- **Мой профиль** — просмотр данных, дети, активные купоны.
- **Редактирование профиля** — изменение имени, данных детей, управление подпиской на рассылки.
- **Купоны** — единоразовые денежные бонусы с ограничением суммы и срока действия.
- **Связаться с продавцом** — быстрый выход на поддержку магазина.

### Для продавцов

- **Регистрация через контакт-карту** — владелец регистрирует нового продавца, переслав его контакт в бота.
- **Найти профиль покупателя** — поиск по номеру клиента (после сканирования QR или ввода вручную).
- **Просмотр профиля** — скидка, активные купоны, список детей.
- **Погашение купона** — кнопка прямо в профиле покупателя, подтверждение одним тапом.

### Для владельца (суперпользователь)

- **Управление персоналом** — список продавцов, добавление через контакт-карту, смена режима (покупатель ↔ продавец).
- **Изменение скидки** — редактирование персонального процента скидки для любого покупателя.
- **Купоны** — выдача купонов покупателям (сумма, процент от чека, срок действия).
- **Рассылки** — создание массовых сообщений с фильтрацией по сегментам: день рождения, пол, наличие детей. Запуск немедленно или по расписанию. Возможность массовой выдачи купонов.
- **Список рассылок** — просмотр запланированных и активных рассылок.
- **Экспорт в Excel** — выгрузка базы покупателей.

### Автоматика (фоновые задачи)

- **Напоминания о днях рождения** — за день до дня рождения ребёнка покупатель получает купон и поздравление.
- **Истечение купонов** — просроченные купоны автоматически деактивируются.
- **Отложенные рассылки** — бот сам отправляет запланированные рассылки в заданное время с задержкой между сообщениями, чтобы не попасть под ограничения API.

## Стек

| Слой | Технология |
|------|------------|
| Бот | `maxapi` (SDK для Max messenger) |
| Язык | Python 3.11+ async |
| БД | PostgreSQL 14+ через SQLAlchemy 2.x async + asyncpg |
| FSM | `maxapi.context.MemoryContext` (in-memory) |
| Конфиг | `python-dotenv` + `.env` |

## Быстрый старт

### Локально

```bash
# 1. Установить зависимости
pip install -e .

# 2. Настроить окружение
cp .env.example .env
# Заполнить .env (см. переменные окружения ниже)

# 3. Применить миграции БД
python scripts/migrate.py

# 4. Запустить бота
python main.py
```

### Docker

```bash
cp .env.example .env
# Заполнить .env

docker compose up -d
```

Первый запуск автоматически применяет миграции и запускает бота.

## Переменные окружения

| Переменная | Описание | Обязательная |
|-----------|----------|:---:|
| `BOT_TOKEN` | Токен бота Max | ✓ |
| `DB_HOST` | Хост PostgreSQL | ✓ |
| `DB_PORT` | Порт PostgreSQL (5432) | |
| `DB_NAME` | Имя базы данных | ✓ |
| `DB_USER` | Пользователь БД | ✓ |
| `DB_PASSWORD` | Пароль БД | ✓ |
| `DISCOUNT_PERCENT` | Размер скидки при регистрации (%) | ✓ |
| `OWNER_ID` | ID аккаунта Max Messenger владельца магазина — при первом запуске `init_db.py` создаёт запись суперпользователя | ✓ |
| `DEEPLINK_BASE` | URL бота для ссылок в QR-кодах (`https://max.ru/botname`) | ✓ |
| `INVITE_SECRET` | Секрет для HMAC-подписи токенов приглашения продавцов | ✓ |
| `TZ` | Часовой пояс бота (`Asia/Yekaterinburg`) | |
| `BROADCAST_SEND_DELAY_SECONDS` | Задержка между сообщениями рассылки в секундах (15) | |
| `BIRTHDAY_COUPON_VALUE` | Сумма купона на день рождения в рублях (300) | |
| `BIRTHDAY_COUPON_VALID_DAYS` | Срок действия купона на день рождения в днях (7) | |

## Структура проекта

```

  handlers/          — обработчики событий Max

specs/               — полные спецификации продукта

  max-botapi.md      — документация по библиотеке maxapi
```

## Разработка

Перед любой работой с кодом читай **AGENTS.md** и спецификации в `specs/`.

Каждый сценарий описан в `specs/scenarios/`. Сущности — в `specs/entities/`. Нефункциональные требования — в `specs/nfr/`.

---

## О подходе к разработке

Код бота сгенерирован AI-агентами (Claude) на основе **Spec Driven Development** —
авторской интерпретации подхода, при котором разработка начинается не с кода,
а со спецификаций в `specs/`. Агент читает спеки, задаёт вопросы, уточняет поведение,
и только после этого генерирует код. Никакого «написал и молись» — каждый сценарий
описан до строчки кода.

Подход позволяет поддерживать живую, актуальную документацию и итерировать продукт
без накопления технического долга в понимании системы.
